# lab_stages Review Checklist

Use this checklist when the task is more about readability than about adding a new
conceptual diagram.

## Naming

- prefer verbs for actions
- prefer semantic names over generic `run trainer` labels
- remove avoidable redundancy in step names

## Flow

- confirm execution order is intelligible
- make install versus run steps obvious
- make outputs comparable and traceable

## Truthfulness

- do not claim conceptual stages that the implementation does not have
- keep semantic labels aligned with actual app defaults
