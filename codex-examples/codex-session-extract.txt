› only one lib is necessary isn't it ?


• In view_maps we still need both names because we have to support environments that only provide one or the other. Python ≥3.11 ships tomllib; earlier versions don’t, so we fall back to tomli. Keeping the try … except ModuleNotFoundError ensures the page works across all runtimes that
  AGILab targets. If we drop one of them, either old or new interpreters would break.


› but why a try mechanism ?


• Because the feature we’re importing doesn’t live in a single module across the Python versions we need to support. In 3.11+ the TOML parser is built-in as tomllib; in 3.10 and earlier it only exists in the third-party tomli package. The try block first attempts the standard-library path,
  which is preferred when available. If that import fails, the except ModuleNotFoundError clause falls back to tomli, so we decode the TOML files the same way no matter which interpreter is running. If we removed the try/except, one of those interpreter families would crash at import time.
