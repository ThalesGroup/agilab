## Short-form positioning

### What AGILAB is

AGILAB is a workflow and orchestration layer for reproducible AI/ML work. It gives the same application a clear control path from local UI or CLI entrypoints to packaged workers, distributed execution, service mode, and analysis.

### Why it is different

- It is **agent-friendly**: the repo includes `AGENTS.md`, `.codex/skills`, and Codex workflow helpers.
- It is **free-threaded-aware**: support is explicit when the chosen environment and worker declare compatibility.
- It focuses on **reproducibility and orchestration**, not only experiment tracking or notebooks.

### What not to say

- Do not claim free-threaded Python is always enabled by default.
- Do not describe AGILAB as a full enterprise platform if the user has not yet seen the workflow proof.
- Do not market it as “everything for AI” or “all-in-one MLOps”.
